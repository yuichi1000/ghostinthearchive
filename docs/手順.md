# ghostinthearchive.ai SSL/HTTPS 設定手順

## 現状

| 項目 | 状態 |
|------|------|
| ドメインレジストラ | Spaceship |
| DNS | Google Cloud DNS（`ghostinthearchive-zone`） |
| `admin.ghostinthearchive.ai` | Cloud Run domain mapping + CNAME → `ghs.googlehosted.com.` |
| `ghostinthearchive.ai`（ルート） | **DNS A レコードなし、HTTPS 不可** |
| 公開サイト | Firebase Hosting にデプロイ済みだがカスタムドメイン未設定 |

## 問題

ルートドメイン `ghostinthearchive.ai` に対して:
1. Firebase Hosting のカスタムドメインが設定されていない
2. Cloud DNS に A レコードが存在しない
3. SSL 証明書が発行されていない

## 設定手順

### Step 1: Firebase Hosting にカスタムドメインを追加

Firebase Console または CLI でカスタムドメインを設定する。

#### 方法 A: Firebase Console（推奨）

1. [Firebase Console](https://console.firebase.google.com/) を開く
2. プロジェクト `ghostinthearchive` を選択
3. 左メニュー「Hosting」→「カスタムドメインを追加」
4. `ghostinthearchive.ai` を入力
5. Firebase が指定する **A レコードの IP アドレス**（通常 2 つ）をメモする

#### 方法 B: Firebase CLI

```bash
firebase hosting:channel:deploy live --project ghostinthearchive
firebase hosting:sites:list --project ghostinthearchive
```

> Firebase Console の方が A レコードの IP アドレスを確認しやすいため推奨。

### Step 2: Cloud DNS に A レコードを追加

Firebase が指定した IP アドレスを Cloud DNS に登録する。

#### 方法 A: gcloud CLI

```bash
# Firebase が指定した IP アドレスを A レコードとして追加
# （IP アドレスは Step 1 で Firebase Console に表示されたものを使用）
gcloud dns record-sets create ghostinthearchive.ai. \
  --zone="ghostinthearchive-zone" \
  --type="A" \
  --ttl="300" \
  --rrdatas="<FIREBASE_IP_1>,<FIREBASE_IP_2>" \
  --project="ghostinthearchive"
```

#### 方法 B: Terraform（推奨・IaC 管理）

`terraform/dns.tf` に以下を追加:

```hcl
# A record for root domain -> Firebase Hosting
resource "google_dns_record_set" "root" {
  name         = "${var.domain}."
  type         = "A"
  ttl          = 300
  managed_zone = google_dns_managed_zone.main.name
  rrdatas      = ["<FIREBASE_IP_1>", "<FIREBASE_IP_2>"]
}
```

```bash
cd terraform
terraform plan
terraform apply
```

> **注意**: Firebase が指定する IP アドレスは固定値（`199.36.158.100` 等）だが、Firebase Console で確認した正確な値を使用すること。

### Step 3: ドメイン所有権の確認（必要な場合）

Firebase が TXT レコードによるドメイン所有権の確認を求める場合がある。

```bash
# Firebase が指定した TXT レコードを追加
gcloud dns record-sets create ghostinthearchive.ai. \
  --zone="ghostinthearchive-zone" \
  --type="TXT" \
  --ttl="300" \
  --rrdatas='"<FIREBASE_VERIFICATION_CODE>"' \
  --project="ghostinthearchive"
```

Terraform の場合:

```hcl
resource "google_dns_record_set" "root_txt" {
  name         = "${var.domain}."
  type         = "TXT"
  ttl          = 300
  managed_zone = google_dns_managed_zone.main.name
  rrdatas      = ["<FIREBASE_VERIFICATION_CODE>"]
}
```

### Step 4: SSL 証明書の自動発行を待つ

1. A レコードが正しく設定されると、Firebase が **自動的に SSL 証明書を発行**する
2. 証明書の発行には **数分〜最大 24 時間** かかる
3. Firebase Console の Hosting ページでステータスを確認できる

### Step 5: www サブドメインの設定（任意）

`www.ghostinthearchive.ai` → `ghostinthearchive.ai` へのリダイレクトが必要な場合:

1. Firebase Console で `www.ghostinthearchive.ai` もカスタムドメインとして追加
2. Cloud DNS に CNAME レコードを追加:

```hcl
resource "google_dns_record_set" "www" {
  name         = "www.${var.domain}."
  type         = "CNAME"
  ttl          = 300
  managed_zone = google_dns_managed_zone.main.name
  rrdatas      = ["${var.domain}."]
}
```

## 確認方法

### DNS 伝播の確認

```bash
# A レコードの確認
dig A ghostinthearchive.ai

# 期待する出力: Firebase の IP アドレスが返ること
# ghostinthearchive.ai.  300  IN  A  <FIREBASE_IP_1>
# ghostinthearchive.ai.  300  IN  A  <FIREBASE_IP_2>
```

### HTTPS アクセスの確認

```bash
# SSL 証明書の確認
curl -vI https://ghostinthearchive.ai 2>&1 | grep -E "subject:|issuer:|expire"

# HTTP → HTTPS リダイレクトの確認
curl -I http://ghostinthearchive.ai
# 期待: 301 → https://ghostinthearchive.ai
```

### Firebase Console での確認

Firebase Console > Hosting で以下を確認:
- ドメインのステータスが「接続済み」になっていること
- SSL 証明書のステータスが「アクティブ」になっていること

## 参考: Firebase Hosting が割り当てる一般的な IP アドレス

Firebase Hosting は通常以下の IP アドレスを使用する（確認のため必ず Console で正確な値を取得すること）:

```
199.36.158.100
```

## 作業の流れまとめ

```
Firebase Console でカスタムドメイン追加
    ↓
A レコードの IP アドレスを確認
    ↓
Cloud DNS に A レコード追加（Terraform or gcloud）
    ↓
（必要なら TXT レコードで所有権確認）
    ↓
SSL 証明書の自動発行を待つ（数分〜24時間）
    ↓
https://ghostinthearchive.ai でアクセス確認
```
